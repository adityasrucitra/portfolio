Understanding @Bean in Spring Boot

Published June 2026 · 10 min read · Spring Boot
SPRING BOOT
@Bean
Dependency Injection

Spring Boot uses Dependency Injection to manage application components automatically.

What is @Bean?

@Bean tells Spring that the returned object should become a managed bean inside the application context. 

@Configuration
public class AppConfig {

    @Bean
    public PersonService personService() {
        return new PersonService();
    }

}

Why Use @Bean?

It allows you to register third-party classes or manually configure objects before Spring injects them into other components.

How Spring Uses It

When the application starts, Spring scans configuration classes and creates all beans inside the application context. 

@Service
public class UserService {

    private final PersonService personService;

    public UserService(PersonService personService) {
        this.personService = personService;
    }
}

Summary

Use @Bean whenever you need explicit control over object creation. For most application components, @Component, @Service, and @Repository are usually sufficient.
